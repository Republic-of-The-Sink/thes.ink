How to *actually* get the BPM of songs

Written by xad1561
July 17, 2026

A few days ago I figured out a couple methods for how to pretty quickly get the BPM of a song, and people probably want to know how, so here you go. In no way do I claim to be even remotely an expert on music, so there are probably easier or better ways out there, but I have found these methods to work very well.

Method A: Using Audacity

You'll need to use Audacity for this method, so make sure you have it installed before continuing.

Firstly, you'll want to import your desired song into Audacity by dragging it in.

Next, you need to find two beats to use as a measuring stick. You can do this by looking for two audio "peaks" and making a selection between them.

Zoom in on the end of your selection and note the time where it ends, then do the same for the start of the selection.

Now subtract the number for the start from the number for the end to get the time between beats, then divide 60 by that to get a close estimate of the BPM.
For the song I&aposm using for demonstration, this would be calculated like:
0.4716 - 0.02679 = 0.44481 60 / 0.444881 = 134.88905

The BPM we calculated is very close to the actual BPM. We will now round up to 135 and check using Audacity's Rhythm Track function under Generate at the top toolbar.

This will open a menu where you can customize various aspects of the rhythm track, but for now just punch in the rounded BPM you got in the previous step and make sure it runs for a decent number of measures.

The generated metronome track may not be properly aligned with the song, so move it around as needed to make sure the first beat is properly synced up. If you did everything correctly, the rhythm track should be synced up with the rest of the song. If the metronome desyncs, then you got it wrong. The earlier it desyncs, the more you got it wrong. Remove the old rhythm track and generate new ones at different BPMs to guess and check until you find the correct tempo.

Once you are sure you have the right BPM for your song, you can then remove all audio tracks, change the tempo in the bottom left to the correct BPM, then reimport your song.

Method B: Searching on Osu! (faster but less honorable than finding it yourself)

Osu! is a rhythm game, but we don't exactly care about the whole rhythm game part that much. We do care that people produce custom beatmaps for it, which means that they have already found the exact tempo of a lot of different songs down to the hundredth of a BPM. Instead of Googling the BPM for a song if you can't find it yourself, search for the beatmap on Osu! instead. Seriously, it works a lot better than you would expect.

Most sites that offer the BPMs for songs do a very bad job of actually helping you with the BPM because they are either just plain wrong or only offer it to the nearest whole number, unlike the Osu! beatmap list, where if someone doesn't find the exact beats per minute right they will be beat up within a minute (I might be exaggerating a little with that last part but you get the point).
